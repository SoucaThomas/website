## Design Context

### Users
Dual audience: fellow developers (browsing during work, researching similar projects in embedded/web) and hiring managers/recruiters evaluating technical depth and communication. Reading context is primarily desktop during work hours, with some mobile/evening browsing.

### Brand Personality
Bold, expressive, energetic. Strong technical credibility that doesn't take itself too seriously. Feels like a person with taste, not a template.

### Aesthetic Direction
- **Theme**: Light with dark code blocks — warm, bright surfaces with high-contrast technical sections
- **Typography**: Distinctive display font paired with clean geometric body. Large, punchy headlines.
- **Layout**: Asymmetric where it serves emphasis, dense where information is rich, generous where content needs to breathe
- **Color**: Vermillion/red-orange accent on warm-tinted neutrals — unexpected for a dev portfolio, memorable
- **Motion**: Purposeful, not decorative — page load stagger, smooth transitions on interaction

### Anti-References
- No brutalist thick-border black/white aesthetic
- No generic SaaS landing page patterns (hero metrics, gradient CTAs)
- No overdesigned Dribbble decoration (illustrations, complex animations)
- No "terminal/hacker" aesthetic

### Design Principles
1. **Content carries the weight** — typography and layout do the work, not decoration
2. **Personality through craft** — the details signal taste (font choices, spacing rhythm, color restraint)
3. **Technical credibility without posturing** — show the work, don't perform "developer-ness"
4. **Surprise in restraint** — one bold choice per section, not everything fighting for attention
5. **Readable above all** — especially for the blog, long-form comfort is non-negotiable
